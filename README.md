# ocr_module
[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Модуль OCR для распознавания чеков и платежных документов
Данный модуль предназначен для автоматического распознавания и извлечения структурированной информации из изображений чеков, счетов, квитанций PayPal и других финансовых документов. Он использует OCR (оптическое распознавание символов) и эвристические алгоритмы для классификации типа документа и выделения ключевых полей: даты, суммы, номера документа, валюты, статуса платежа и др.

Основные возможности
Классификация документа: определение типа (receipt, invoice, paypal_receipt, unknown) на основе ключевых слов.

# Извлечение полей:

Дата (в различных форматах)

Общая сумма (с поддержкой разных валют и разделителей)

Номер документа/операции

Валюта (RUB, USD, EUR)

Платёжная система (СБП, PayPal)

Статус платежа (completed/pending)

# Специализированные методы:

Для российских чеков (включая операции через СБП)

Для PayPal-квитанций (извлечение ID транзакции, получателя, суммы в USD)

Эвристики и валидация сумм: поиск суммы по контекстным ключевым словам, проверка реалистичности значений.

Нормализация текста: замена похожих символов для улучшения распознавания.

Экспериментальный анализ размера шрифта: попытка выделить сумму по самому крупному тексту (требуется pytesseract с детальным выводом).

Установка зависимостей

Для полноценной работы рекомендуется установить:

```bash
pip install pytesseract pillow pandas
```
Также потребуется установить Tesseract OCR (системная зависимость) и языковые пакеты rus и eng.

# Использование

Простой пример
```python
from ocr import ocr_extract
```
# Путь к изображению чека или PDF

result = ocr_extract("path/to/receipt.jpg")
print(result["type"])          # тип документа
print(result["fields"])        # извлечённые поля
print(result["text_preview"])  # первые 1000 символов распознанного текста

Асинхронный вариант с анализом шрифта

```python
import asyncio
from ocr import OCRModule

async def main():
    result = await OCRModule.ocr_extract_with_font("receipt.png", use_font_analysis=True)
    print(result)

asyncio.run(main())
```
# Структура возвращаемых данных

Функции ocr_extract и ocr_extract_with_font возвращают словарь:

```python
{
    "type": str,                # "receipt", "invoice", "paypal_receipt", "unknown"
    "fields": dict,              # извлечённые поля (date, total, number, currency, ...)
    "text_preview": str,         # первые 1000 символов текста
    "raw_text": str,             # полный распознанный текст (опционально)
    "font_analysis_used": bool   # флаг использования анализа шрифта
}
```
Поля в fields могут включать:

date – дата документа

total – сумма (строка, приведённая к формату 1234.56)

number – номер документа/операции

currency – валюта (RUB, USD, EUR)

payment_system – платёжная система (СБП, PayPal, Неизвестно)

status – статус платежа (completed, pending)

transaction_id – ID транзакции (для PayPal)

recipient – получатель (для PayPal)

# Примечания
Модуль предполагает, что текст из файла извлекается функцией extract_text из соседнего модуля file_processing. Если этот модуль отсутствует, OCR на PDF и изображениях не будет работать (функции extract_text_from_image и extract_text_from_pdf используют заглушки или требуют pytesseract).

Класс OCRModule содержит как статические методы, так и методы экземпляра – в текущей реализации все методы статические, поэтому экземпляр создавать не обязательно.

Некоторые методы (например, extract_russian_receipt_fields_with_font) являются экспериментальными и могут требовать детального вывода Tesseract.

Для продакшена рекомендуется использовать основной метод ocr_extract (или асинхронную обёртку) и при необходимости дорабатывать под конкретные форматы документов.
